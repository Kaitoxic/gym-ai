<task type='auto' tdd='true'>
  <name>Task 1: JWT auth middleware + quota middleware</name>
  <files>src/middleware/auth.ts, src/middleware/quota.ts</files>
  <behavior>
    auth.ts:
    - Missing Authorization header -> 401 { error: Missing token }
    - Token with wrong signature -> 401 { error: Invalid token }
    - Expired token -> 401 { error: Invalid token }
    - Valid Supabase JWT -> calls next(), req.user = { id, email }
    quota.ts (requireAuth must have run first):
    - ai_calls = 0 -> increments to 1, calls next()
    - ai_calls = 49 -> increments to 50, calls next()
    - ai_calls = 50 -> 429 { error: Daily quota exceeded }, no next()
    - reset_date < today -> resets ai_calls to 1, updates reset_date, calls next()
    - user not in user_quotas -> inserts (ai_calls=1, reset_date=today), calls next()
  </behavior>
  <action>
    Install: npm install jsonwebtoken @types/jsonwebtoken

    src/middleware/auth.ts:
    - Import jwt from jsonwebtoken, RequestHandler from express
    - Export requireAuth: RequestHandler
    - Extract Bearer token from Authorization header; if missing return 401 { error: Missing token }
    - jwt.verify(token, process.env.JWT_SECRET!) -- Supabase uses HS256
    - Cast payload as { sub: string; email: string }
    - req.user = { id: payload.sub, email: payload.email }; next()
    - Catch JsonWebTokenError / TokenExpiredError: 401 { error: Invalid token }

    src/middleware/quota.ts:
    - Import supabase from ../lib/supabase, RequestHandler from express
    - Export checkQuota: RequestHandler (async)
    - today = new Date().toISOString().slice(0, 10) // YYYY-MM-DD
    - Query user_quotas WHERE user_id = req.user!.id
    - No row: insert { user_id, ai_calls: 1, reset_date: today }, next()
    - row.reset_date < today: update ai_calls=1, reset_date=today, next()
    - row.ai_calls >= 50: return 429 { error: Daily quota exceeded }
    - else: update ai_calls = row.ai_calls + 1, next()
    - NOT atomic -- acceptable for v1 (document in comment)
    - try/catch: 500 on unexpected DB error
  </action>

  <files>src/middleware/validateBody.ts, src/services/aiRouter.ts</files>
  <behavior>
    validateBody(schema):
    - Body passes schema -> calls next()
    - Body fails schema -> 400 { error: Validation failed, details: ZodError.errors }
    - Body undefined/null -> 400

    callAI(provider, model, messages):
    - provider=openrouter -> openai SDK with baseURL https://openrouter.ai/api/v1 + OPENROUTER_API_KEY, returns string
    - provider=openai -> openai SDK default baseURL + OPENAI_API_KEY, returns string
    - provider=gemini -> @google/generative-ai, converts messages, returns string
    - unknown provider -> throws Error(Unsupported provider: X)
    - API errors propagate to caller
  </behavior>
  <action>
    Install: npm install zod openai @google/generative-ai

    src/middleware/validateBody.ts (implement stub from Plan 01-01):
    - Import ZodSchema from zod, RequestHandler from express
    - Export validateBody(schema: ZodSchema): RequestHandler
    - result = schema.safeParse(req.body)
    - if !result.success: 400 { error: Validation failed, details: result.error.errors }
    - if success: req.body = result.data; next()

    src/services/aiRouter.ts:
    - type Message = { role: user | assistant | system; content: string }
    - type Provider = openrouter | openai | gemini
    - export async callAI(provider, model, messages): Promise<string>
    - openrouter: new OpenAI({ apiKey: process.env.OPENROUTER_API_KEY, baseURL: https://openrouter.ai/api/v1 })
      -> chat.completions.create({ model, messages }) -> choices[0].message.content ?? empty string
    - openai: new OpenAI({ apiKey: process.env.OPENAI_API_KEY }) -> same pattern
    - gemini:
        const genAI = new GoogleGenerativeAI(process.env.GEMINI_API_KEY!)
        const gemModel = genAI.getGenerativeModel({ model })
        history = messages.slice(0,-1).filter(m => m.role !== system).map(m => ({
          role: m.role === assistant ? model : user, parts: [{ text: m.content }]
        }))
        const chat = gemModel.startChat({ history })
        const r = await chat.sendMessage(messages[messages.length-1].content)
        return r.response.text()
    - default: throw new Error(Unsupported provider: + provider)
    - Instantiate SDK clients inside function body (not module-level) -- missing env vars fail at call time

  <files>src/routes/ai/proxy.ts, src/index.ts</files>
  <action>
    src/routes/ai/proxy.ts:
    - Import Router from express, z from zod
    - Import requireAuth, checkQuota, validateBody, callAI
    - ProxySchema = z.object({
        provider: z.enum([openrouter, openai, gemini]),
        model: z.string().min(1),
        messages: z.array(z.object({ role: z.enum([user, assistant, system]), content: z.string() })).min(1)
      })
    - const router = Router()
    - router.post(/, requireAuth, checkQuota, validateBody(ProxySchema), async (req, res) => {
        try {
          const { provider, model, messages } = req.body
          const result = await callAI(provider, model, messages)
          res.json({ result })
        } catch (err: any) {
          res.status(502).json({ error: AI provider error, detail: err.message })
        }
      })
    - export default router

    src/index.ts update:

    - app.use(/ai, aiRouter)  // after express.json(), before error handler
